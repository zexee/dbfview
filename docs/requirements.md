# dbfview 需求文档

## 1. 项目概述

**项目名称**: dbfview — DBF 文件查看/编辑器

**目标**: 轻量级 Windows 桌面工具，用于打开、查看、编辑和保存 DBF 2000 格式文件，以表格形式呈现数据，支持简单过滤。

**运行环境**: Windows 10/11, .NET 9 Desktop Runtime

---

## 2. 功能需求

### 2.1 文件操作

| ID | 功能 | 描述 |
|----|------|------|
| F1 | 打开文件 | 菜单/工具栏打开 `.dbf` 文件，打开时选择编码（UTF-8 / GBK） |
| F2 | 保存文件 | 保存当前数据到原文件，使用打开时的编码 |
| F3 | 命令行启动 | 支持 `dbfview.exe "path\to\file.dbf"` 直接打开，以便注册为 `.dbf` 默认打开程序 |

### 2.2 表格显示与编辑

| ID | 功能 | 描述 |
|----|------|------|
| T1 | 表格展示 | DataGridView 以行/列形式展示所有字段和记录 |
| T2 | 单元格编辑 | 直接在单元格内修改数据，C 型字段为文本，N 型字段仅允许数字和小数点 |
| T3 | 删除标记 | Ctrl+Delete 切换当前行删除标记，已删除行灰色显示 |
| T4 | 列宽拖拽 | 鼠标拖拽列分隔线调整列宽（DataGridView 原生支持） |
| T5 | 适配内容 | 一键将所有列宽设置为该列最宽内容的宽度 |
| T6 | 适配窗口 | 一键让所有列等宽填满当前窗口可视区域 |

### 2.3 过滤功能

| ID | 功能 | 描述 |
|----|------|------|
| FL1 | 表达式过滤 | 输入框输入条件（如 `A=123` 或 `NAME LIKE '张%'`），回车或按钮应用 |
| FL2 | 清除过滤 | 一键清除过滤条件，恢复显示全部记录 |
| FL3 | 过滤语法 | 支持 = / > / < / >= / <= / <> / LIKE，支持 AND / OR 组合，字符串字面量用单引号 |
| FL4 | 状态提示 | 状态栏显示"已筛选 N 条 / 共 M 条" |

### 2.4 编码支持

| ID | 功能 | 描述 |
|----|------|------|
| E1 | 自动检测 | 打开文件时自动检测编码（检测 UTF-8 BOM + 字段名合法性试探），检测结果显示在下拉框中 |
| E2 | 手动选择 | 编码下拉框支持手动切换（UTF-8 / GBK），切换后即时重读当前文件 |
| E3 | 保存时使用 | 保存文件使用当前选择的编码 |

---

## 3. 非功能需求

| 编号 | 内容 |
|------|------|
| NF1 | 文件格式 DBF 2000（版本字节 0x30），兼容 dBase III/IV |
| NF2 | 字段类型仅支持编辑 C（字符）和 N（数值），其他类型只读 |
| NF3 | 10 万行以内流畅操作，不做分页 |
| NF4 | 界面语言为中文 |
| NF5 | 单窗口应用，无多文档界面 |

## 5. 自动编码检测策略

打开文件时按以下顺序自动检测编码：

1. 读取 DBF 文件头部（前 32 + 字段区字节）
2. 检查是否存在 UTF-8 BOM（EF BB BF），有则判定 UTF-8
3. 将字段名区域按 UTF-8 解码，如有非法字节序列则判定为 GBK
4. 读取 DBF 头字节 29 的 Language Driver 字段辅助判断
5. 以上均无法判断时默认 GBK

检测结果自动设置到编码下拉框，用户可随时手动切换。
